# 가계부 앱 디자인 문서

## 개요

본 문서는 소셜 로그인과 클라우드 동기화를 지원하는 가계부 모바일 애플리케이션의 UI/UX 디자인 방향을 정의합니다. 모든 디자인은 **모바일 세로 방향(9:16)**과 **한 손 사용**을 전제로 하며, **Apple Human Interface Guidelines (HIG)**를 준수하여 네이티브 iOS 앱과 동일한 사용자 경험을 제공합니다.

## 디자인 원칙

- **단순함과 명확성:** 사용자가 직관적으로 수입과 지출을 기록하고 확인할 수 있도록 합니다.
- **일관성:** iOS 네이티브 앱의 디자인 패턴을 따라 사용자에게 익숙한 인터페이스를 제공합니다.
- **접근성:** 큰 터치 영역, 명확한 레이블, 충분한 대비를 통해 모든 사용자가 쉽게 사용할 수 있도록 합니다.
- **시각적 피드백:** 모든 인터랙션에 즉각적인 피드백(햅틱, 애니메이션)을 제공합니다.

## 화면 구성

### 1. 로그인 화면 (Login Screen)

**목적:** 사용자가 소셜 계정을 통해 간편하게 로그인합니다.

**주요 콘텐츠:**
- 앱 로고 및 타이틀
- 소셜 로그인 버튼 (Google, Apple)
- 간단한 설명 문구

**레이아웃:**
- 화면 상단 1/3: 앱 로고와 타이틀
- 화면 중앙: 소셜 로그인 버튼 (세로로 배치)
- 화면 하단: 개인정보처리방침 링크

**주요 사용자 플로우:**
1. 사용자가 앱을 실행합니다.
2. 로그인 화면에서 소셜 로그인 버튼을 탭합니다.
3. 인증 후 홈 화면으로 이동합니다.

---

### 2. 홈 화면 (Home Screen)

**목적:** 사용자의 현재 재정 상태를 한눈에 보여주고, 빠르게 수입/지출을 추가할 수 있도록 합니다.

**주요 콘텐츠:**
- 이번 달 총 수입/지출 요약 카드
- 최근 거래 내역 리스트 (최대 10개)
- 하단에 고정된 "추가" 버튼 (FAB)

**레이아웃:**
- 상단: 월별 요약 카드 (수입, 지출, 잔액)
- 중앙: 최근 거래 내역 리스트 (스크롤 가능)
- 하단 우측: 플로팅 액션 버튼 (+ 아이콘)

**주요 사용자 플로우:**
1. 사용자가 홈 화면에서 월별 요약을 확인합니다.
2. 최근 거래 내역을 스크롤하여 확인합니다.
3. "+" 버튼을 탭하여 새로운 거래를 추가합니다.

---

### 3. 거래 추가/수정 화면 (Add/Edit Transaction Screen)

**목적:** 사용자가 수입 또는 지출 내역을 입력하거나 수정합니다.

**주요 콘텐츠:**
- 거래 유형 선택 (수입/지출 토글)
- 금액 입력 필드 (숫자 키패드)
- 카테고리 선택 (드롭다운 또는 모달)
- 날짜 선택 (기본값: 오늘)
- 메모 입력 필드 (선택 사항)
- 저장 버튼

**레이아웃:**
- 상단: 거래 유형 토글 (수입/지출)
- 중앙: 금액 입력 필드 (큰 폰트)
- 하단: 카테고리, 날짜, 메모 입력 필드
- 최하단: 저장 버튼

**주요 사용자 플로우:**
1. 사용자가 홈 화면에서 "+" 버튼을 탭합니다.
2. 거래 추가 화면에서 유형을 선택합니다.
3. 금액, 카테고리, 날짜, 메모를 입력합니다.
4. "저장" 버튼을 탭하여 거래를 저장합니다.
5. 홈 화면으로 돌아갑니다.

---

### 4. 통계 화면 (Statistics Screen)

**목적:** 사용자의 수입/지출 패턴을 시각적으로 분석하여 보여줍니다.

**주요 콘텐츠:**
- 월별 수입/지출 추이 차트 (라인 차트)
- 카테고리별 지출 비율 (파이 차트)
- 상위 지출 카테고리 리스트

**레이아웃:**
- 상단: 기간 선택 (이번 달, 지난 달, 최근 3개월)
- 중앙: 차트 영역 (스크롤 가능)
- 하단: 상위 카테고리 리스트

**주요 사용자 플로우:**
1. 사용자가 탭 바에서 "통계" 탭을 선택합니다.
2. 기간을 선택하여 원하는 기간의 통계를 확인합니다.
3. 차트를 스크롤하여 상세 정보를 확인합니다.

---

### 5. 설정 화면 (Settings Screen)

**목적:** 사용자가 앱 설정을 변경하고 계정을 관리합니다.

**주요 콘텐츠:**
- 사용자 프로필 정보
- 카테고리 관리 (추가, 수정, 삭제)
- 테마 설정 (라이트/다크 모드)
- 로그아웃 버튼

**레이아웃:**
- 상단: 사용자 프로필 카드
- 중앙: 설정 항목 리스트
- 하단: 로그아웃 버튼

**주요 사용자 플로우:**
1. 사용자가 탭 바에서 "설정" 탭을 선택합니다.
2. 원하는 설정 항목을 탭하여 변경합니다.
3. 로그아웃 버튼을 탭하여 로그아웃합니다.

## 내비게이션 구조

앱은 **탭 바 내비게이션**을 사용하여 주요 화면 간 이동을 지원합니다.

| 탭 | 아이콘 | 화면 |
|---|---|---|
| **홈** | `house.fill` | 홈 화면 |
| **통계** | `chart.bar.fill` | 통계 화면 |
| **설정** | `gearshape.fill` | 설정 화면 |

## 주요 인터랙션

- **거래 추가:** 홈 화면의 플로팅 액션 버튼(+)을 탭하여 거래 추가 화면으로 이동합니다.
- **거래 수정/삭제:** 홈 화면의 거래 항목을 탭하여 상세 화면으로 이동하고, 수정 또는 삭제를 수행합니다.
- **카테고리 관리:** 설정 화면에서 카테고리를 추가, 수정, 삭제합니다.
- **통계 확인:** 통계 탭에서 기간을 선택하여 차트를 확인합니다.

---

## 접근성 고려사항

- **터치 영역:** 모든 버튼과 인터랙티브 요소는 최소 44x44pt 크기를 유지합니다.
- **대비:** 텍스트와 배경 간 충분한 대비를 유지하여 가독성을 높입니다.
- **햅틱 피드백:** 주요 액션(버튼 탭, 저장 완료)에 햅틱 피드백을 제공합니다.
- **다크 모드:** 라이트 모드와 다크 모드 모두에서 일관된 사용자 경험을 제공합니다.

---

## 기술적 고려사항

- **데이터 동기화:** 모든 거래 데이터는 클라우드 데이터베이스에 실시간으로 동기화됩니다.
- **오프라인 지원:** 네트워크 연결이 없을 때도 로컬에 데이터를 저장하고, 연결 복구 시 자동 동기화합니다.
- **보안:** 사용자 인증은 OAuth를 통해 안전하게 처리되며, 모든 데이터는 암호화되어 전송됩니다.
